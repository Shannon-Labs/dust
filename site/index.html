What normal databases can't do

Database branching

Create branches like git. Experiment with schema changes without touching production. dust branch create dev

Postgres wire protocol

dust serve starts a server that speaks Postgres. Connect with psql, DataGrip, any ORM. Your existing tools just work.

Schema fingerprinting

Every schema state gets a BLAKE3 fingerprint in dust.lock. Detect drift, validate migrations, never deploy stale.

Full SQL engine

SELECT with WHERE, JOINs, GROUP BY, ORDER BY, LIMIT. Aggregates: COUNT, SUM, AVG, MIN, MAX. INSERT, UPDATE, DELETE.

Built-in health checks

dust doctor validates your schema, lockfile, fingerprints, and workspace integrity in one command.

Zero dependencies

One 8MB binary. No Docker. No JVM. No runtime. Works offline. curl | sh and you're done.

$ curl -fsSL https://dustdb.dev/install.sh | sh
