What you can do today

Dust is already useful when the job is local development, branchable database state, deterministic seed data, or agent-visible SQL workflows. It is deliberately not claiming production-database maturity.

Fast local loop

`dust init`, `dust query`, `dust shell`, `dust doctor`, and `dust demo` cover the first-run path without Docker or background services.

Branchable state

`dust branch`, `dust snapshot`, `dust diff`, and `dust merge` make database state explicit and scriptable in the repo.

Agent surfaces

Dust exposes the same project over pgwire, MCP, and a thin Python client built over JSON CLI output.
