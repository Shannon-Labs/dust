Support

Dust now routes launch traffic through repo-owned public issue forms instead of relying on private messages or ad hoc email.

Public Surfaces

  • Support question for workflow and onboarding help.
  • Bug report for reproducible defects.
  • Docs feedback for missing, incorrect, or unclear docs.
  • Team beta intake for invite-only beta qualification.
  • Enterprise contact for design-partner or higher-touch rollout conversations.
  • Feature request for product direction requests.

Ownership And Escalation

  • Public questions should be answered in the issue thread so the resolution becomes reusable.
  • Reproducible defects should turn into bug issues with clear reproduction steps and version context.
  • Docs feedback should be folded back into the docs tree quickly so launch traffic sees the fix.
  • Team beta and enterprise intake remain manual operator queues until dedicated CRM and billing automation exist.
  • Severe blockers, security-sensitive material, or account-specific conversations can move off-thread after the initial public intake is logged.

Why This Exists

  • Launch traffic needs a visible support path.
  • Repeated questions should become searchable public answers.
  • The project should not imply a staffed helpdesk or hosted SaaS support model that does not exist yet.
